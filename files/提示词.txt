### 模组开发提示词（最终版）：Minecraft 可编程频道总线单片机系统

**模组名称**：Channel Bus Microcontroller (频道总线单片机)  
**目标版本**：Minecraft 1.20+（可指定）  
**模组平台**：Fabric（推荐，需 Fabric API）或 Forge  
**核心玩法**：  
在 Minecraft 中搭建由线缆、IO 接口和单片机组成的频道总线网络。单片机可通过**内置图形化编程界面**编写本地控制逻辑，或通过**外部程序接口（TCP Socket）** 连接 Python 等外部程序进行实时控制。外部接口与本地编程可共存，并允许互相切换。

---

#### 一、总体架构

模组分为六大于系统：
1.  **线缆网络系统**：方块间逻辑连接与频道数据共享。
2.  **IO 接口方块**：红石信号的物理输入/输出端点，可配置模式和绑定频道。
3.  **单片机方块**：核心处理单元，包含：
    - 频道输入聚合与输出分发
    - 本地图形化编程引擎
    - 外部程序通信接口（TCP Socket）
4.  **图形化编程系统**：集成在单片机 GUI 中，基于节点图的可视化编程。
5.  **外部通信模块**：基于 TCP Socket 的双向 JSON 协议，保留原外部 Python 接口。
6.  **工具与调试**：网络可视化、频道监视器等。

---

#### 二、方块与物品设计

##### 1. 线缆方块 (Cable Block)
- **变种**：8 频道线缆 (基础)、16 频道线缆 (中级)、32 频道线缆 (高级)
- **功能**：物理连接载体，不存储数据，只参与组网。
- **模型**：自动根据相邻线缆、IO 接口、单片机生成连接模型（使用 Multipart 或自定义模型，参考原版红石粉连接逻辑）。
- **配方**：不同等级使用不同材料（如铜、金、钻石等）。

##### 2. IO 接口方块 (IO Interface Block)
- **方块实体**：存储 6 个面的配置。
- **每个面可配置项**：
  - `mode`：`INPUT` 或 `OUTPUT`
  - `channel`：绑定的频道号 (1 ~ 当前线缆支持的最大频道数)
- **行为**：
  - 输入模式：每 tick 读取本面朝向的红石信号，将强度写入频道网络（冲突时取最大值）。
  - 输出模式：每 tick 从频道网络读取对应频道强度，在对应面输出红石信号。
- **红石交互**：覆写 `getWeakRedstonePower()` 动态输出；用 `world.getRedstonePower(pos, direction)` 读取。
- **GUI**：右键打开，6个面独立配置（模式下拉、频道滑块），实时显示各面状态。

##### 3. 单片机方块 (Microcontroller Block)
- **方块实体**：核心复杂实体，包含以下组件：
  - 6 个面的端口配置
  - 本地图形化程序编译实例
  - 外部 Socket 客户端
- **每个面的配置项（端口）**：
  - `mode`：`INPUT_PORT` 或 `OUTPUT_PORT`
  - `channel_start`、`channel_count`：占用频道范围
  - `label`：唯一标签（用于图形化编程节点和 JSON 标识）
- **GUI 功能分区**（通过标签页切换）：
  1. **端口配置**页：配置 6 个面的模式、频道范围、标签。
  2. **编程**页：图形化编程画布（见下文）。
  3. **外部接口**页：配置 TCP 连接（IP、端口）、连接状态、模式切换。
  4. **仪表板**页（可选）：实时显示所有端口频道值。
- **工作模式**：
  - **本地模式**：只运行图形化程序，忽略外部命令。
  - **外部模式**：接收外部 Python 命令并执行，本地程序被禁用（或可设为本地预处理后转发外部，但默认二选一）。
  - 可通过 GUI 按钮切换。
- **核心执行循环**（每 5 tick 或其他可配置间隔）：
  1. 从频道网络读取所有 `INPUT_PORT` 范围的频道值，放入输入寄存器。
  2. 若处于**本地模式**，执行图形化程序，程序访问输入寄存器并修改输出寄存器。
  3. 若处于**外部模式**，将当前输入寄存器数据打包 JSON 发送给外部 Python；等待外部返回命令并更新输出寄存器。
  4. 最后，将输出寄存器的值写入对应的 `OUTPUT_PORT` 频道范围（写入网络）。
- **加载外部程序接口**：除 Socket 通信外，可考虑提供“导入本地脚本”功能，即从文件加载 Lua 脚本或预编译字节码（可选扩展，但必须保留 TCP Socket 接口）。

#### 三、线缆网络系统设计

（此部分保持不变，与上次提示词一致）  
- 网络发现使用 BFS/Flood Fill，当放置/破坏相关方块时触发。
- 每个连通组件分配 `networkId`，全局 `NetworkManager` 管理。
- `ChannelNetwork` 持有 `channelValues` 映射 (channel -> strength) 和最大频道数。
- 输入写入时取最大值，输出写入直接覆盖，防止循环通过标记处理。

---

#### 四、图形化编程系统设计（本地模式核心）

##### 4.1 设计原则
- 采用**数据流节点图**范式，类似 Blender 材质节点、Unreal Engine Blueprint 或 Pure Data。
- 玩家在画布上放置节点，通过连线连接节点的输入/输出端口。
- 图表描述数据如何从**输入端口节点**流向一系列处理节点，最终到达**输出端口节点**。
- 执行时每 tick（或每次评估）对图表进行一次求值，信号为整数（0-15）。

##### 4.2 节点类型
节点分为三大类：**IO节点**、**运算节点**、**控制节点**。

**IO节点**（与配置的端口标签绑定）：
- 输入节点：代表一个 `INPUT_PORT`，输出该端口的频道值数组（多个输出引脚对应频道，或输出一个列表）。
  - 示例节点：“北侧传感器 (`north_port`)” 输出 8 个频道值的单独引脚或打包列表。
- 输出节点：代表一个 `OUTPUT_PORT`，接收多个输入值并映射到对应频道。
  - 示例节点：“南侧驱动 (`south_port`)” 具有 4 个输入引脚，写入频道。

**运算节点**（处理单值）：
- 常数 (0-15)
- 算术：加、减、乘、除（限幅到0-15）
- 比较：大于、小于、等于（输出 0 或 15）
- 逻辑：与、或、非、异或（按位，将强度视为4位二进制）
- 映射/缩放：将输入范围映射到输出范围
- 延迟/保持：上一次输入值

**控制节点**：
- 选择器/多路复用器：根据控制值选择输入 A 或 B。
- 触发器/RS锁存器：基于置位/复位信号。
- 计数器/累加器：脉冲计数。
- 比较器带死区：防止抖动。
- 自定义函数节点（可选扩展，允许玩家定义简单数学表达式）。

##### 4.3 GUI 交互
- 编程画布位于单片机 GUI 的“编程”选项卡内，占据整个右侧区域，可滚轮缩放，拖拽平移。
- 左侧提供节点工具箱，按类别排列，可拖拽到画布。
- 节点具有明确类型的输入/输出引脚（小圆点），颜色区分数据类型（红石强度、布尔、数组）。
- 连线：点击并拖拽从一个引脚到另一个引脚建立连接，非法连接（类型不匹配或循环）应禁止并给出视觉反馈。
- 每个 IO 节点自动与已配置的端口标签关联（不可删除或复制，每标签只一个输入/一个输出节点）。
- 可添加注释框。

##### 4.4 编译与执行模型
- 在画布上编辑时，实时进行拓扑排序检查合法性（有向无环图 DAG）。
- 保存图形时生成一个执行顺序列表（拓扑顺序）。
- 每次单片机执行周期：
  1. 清空所有节点的输入缓存。
  2. 从输入节点开始，按拓扑顺序计算每个节点（将输入引脚值传递给节点函数，计算输出并推送到相连节点的输入）。
  3. 运算节点的计算结果缓存在本周期内，防止重复计算。
  4. 计算完成后，读取输出节点的值，写入输出寄存器。
- 若图形非法（循环、缺少输入连接等），在 GUI 中高亮错误，并拒绝执行，保持上次输出或默认 0。

##### 4.5 与外部程序接口的关系
- 当切换至“外部模式”时，图形化程序被暂停，端口数据直接透传给外部。
- 保留“外部模式”下的原始 TCP Socket 协议不变。
- 未来可扩展“混合模式”：图形化程序预处理输入数据后发送给外部程序，但暂不要求。

#### 五、外部通信协议（TCP Socket + JSON）

（与上次一致，保持不变）
- 单片机作为客户端，连接到外部程序（如 Python 本地服务端）。
- 上行（Minecraft → Python）：`{"type":"input_report","timestamp":...,"data":{"port_label":[...]}}`
- 下行（Python → Minecraft）：`{"type":"output_command","data":[{"target":"port_label","values":[...]}]}`
- 数据帧以 `\n` 分隔。
- 外部模式激活时使用此协议；若连接断开，输出保持或归零可配置。

---

#### 六、线程安全与性能

- 图形化程序执行必须在服务端主线程，因为操作频道网络。
- 外部通信的 Socket 发送使用独立线程池异步进行，接收线程收到消息后投递到主线程执行。
- 图形节点计算需轻量，避免复杂循环。

---

#### 七、开发迭代建议

1. **基础方块与网络**：线缆、IO接口、单片机基础方块和网络系统，先实现频道数据共享。
2. **IO接口与红石交互**：完成输入输出配置和红石交互，测试频道网络。
3. **单片机端口与外部通信**：实现端口配置、数据聚合分发和 TCP Socket 通信，完成外部 Python 控制闭环。
4. **图形化编程框架**：在单片机 GUI 内实现节点画布、节点基础类、连线交互，先只支持常数和直通。
5. **运算节点与控制节点**：逐步添加各类运算节点，完善执行引擎。
6. **集成与模式切换**：实现本地模式和外部模式的切换逻辑，整合所有功能。
7. **优化、美化与扩展**：线缆模型、GUI 美化、配方、手册、文件导入接口等。
